# 质量评审工作流

## 适用场景
需求分析和本体建模完成后，需要对需求完整性、一致性、可追溯性进行质量评审。

## 评审维度
- 完整性: 是否覆盖主流程、异常流程、边界条件、非功能要求
- 一致性: 文档、流程、页面、接口、数据字典是否一致
- 可追溯性: 每个需求是否可追溯到来源文档、规则或用户确认
- 可实现性: 系统边界、接口、数据来源和依赖条件是否明确
- 可测试性: 是否有验收标准、测试用例和覆盖关系

## 工作流
1. 调用 governance-mcp.match_rules 检查规则完整性
2. 调用 governance-mcp.quality_review 计算各维度评分
3. 生成评审报告: 评分卡 + 问题清单 + 风险项 + 待确认项
4. 输出: 评审记录 (ReviewRecord)

## 就绪评分公式
开发就绪评分 = 完整性×20% + 一致性×15% + 可追溯性×15% + 可实现性×15% + 可测试性×15% + 安全合规性×10% + 可复用性×10%

## 约束
- AI 评审结果必须包含评分依据和来源引用
- 评分 < 80% 标记为"需整改"
- 高风险问题必须经人工确认
