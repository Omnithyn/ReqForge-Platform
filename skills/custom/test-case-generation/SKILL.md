# 测试用例生成工作流

## 适用场景
需求分析完成、业务规则确定后，需要生成测试用例。

## 覆盖类型
- 正常流程: 标准业务操作的验证
- 异常流程: 错误输入和异常操作的验证
- 边界条件: 字段边界值、极限值的验证
- 权限校验: 不同角色权限的验证
- 数据校验: 字段格式、必填、范围的验证

## 工作流
1. 读取所有已确认的业务规则
2. 对每条规则生成 3 类测试用例（正常/异常/边界）
3. 标注: 用例编号、场景类型、覆盖规则、优先级(P0/P1/P2)
4. 关联: 测试用例 ↔ 业务规则 ↔ 接口
5. 输出: 测试用例列表 + 覆盖矩阵

## 约束
- P0 规则必须 100% 覆盖
- 每条测试用例必须包含: 前置条件、测试步骤、预期结果、验收标准
