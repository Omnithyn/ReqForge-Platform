# 研发准备包打包工作流

## 适用场景
需求分析、本体建模、质量评审全部通过后，需要输出可进入开发的完整研发准备包。

## 成果物清单
- 需求说明书 (PRD): 背景、目标、范围、功能需求、非功能需求、约束
- 业务流程图: 主流程、异常流程、状态流转
- 页面原型说明: 页面结构、字段布局、交互规则
- 数据字典: 业务对象、字段名称、类型、来源、校验规则
- OpenAPI 草案: 请求参数、响应结构、错误码
- 业务规则清单: 编号、描述、来源依据
- 测试用例: 正常/异常/边界 + 验收标准
- 追溯矩阵: 来源文档→需求→规则→字段→接口→测试
- 研发任务拆解: 模块、接口、页面、后端服务、测试任务
- 风险与待确认项

## 工作流
1. 收集所有已完成评审的中间产物
2. 调用 evidence-mcp.generate_evidence 建立追溯链
3. 调用 mermaid-mcp.generate_flowchart 生成最终流程图
4. 按模板格式打包
5. 标注版本号和生成时间

## 约束
- 所有成果物必须带来源引用
- 待确认项必须列出
- 输出版本号 v{major}.{minor}.{patch}
