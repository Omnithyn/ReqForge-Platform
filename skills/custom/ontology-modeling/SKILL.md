# 本体建模工作流

## 适用场景
需求分析完成后，需要建立业务对象、属性、关系、规则的结构化本体模型时使用。

## 工作流
1. 读取需求分析结果（功能需求、非功能需求、业务规则）
2. 调用 ontology-mcp.map_event_to_ontology 建立事件-对象映射
3. 识别: 业务主体、业务对象、业务流程、数据字段、接口服务
4. 调用 ontology-mcp.establish_facts 建立对象间关系
5. 检查: 继承/扩展/覆盖 全局本体
6. 输出: 本体对象列表 + 关系图 + 追溯关系

## 约束
- 项目专属本体必须标注与全局本体的关系（继承/扩展/覆盖）
- 覆盖全局本体必须标注原因
- 新建对象类型需提交人工确认
